# コンボリファレンス

`config/cornix.keymap` で定義されている全コンボの一覧です。

全コンボは idle ゲート `require-prior-idle-ms = 175` (`COMBO_IDLE`) を共有し、これが打鍵流中のコンボを無効化します。同時押しの窓 `timeout-ms` は指のジオメトリで **三段階**にしています: 同指の縦ペアは `30` (`COMBO_TERM`)、別指の横並びペアは `60` (`COMBO_TERM_HORIZ`)、最もスパンの広い index+小指の Enter (J + ;) は `75` (`COMBO_TERM_WIDE`)。根拠は [コンボ誤爆対策チューニング](#コンボ誤爆対策チューニング) を参照。

## 設計思想

このキーマップは **urob 流「timeless」ホームロウ Mods (HRM)** と多数のコンボを組み合わせた設計です：

- **修飾キー** (Shift / Ctrl / Alt / GUI) は home 行のホームロウ Mods で取得します。下記 [ホームロウ Mods](#ホームロウ-mods) を参照。
- **編集系キー** (Esc / Tab / Delete / Backspace / Insert / Enter)、**リーダーキー**、および **記号・括弧** の大半はコンボで取得します。

HRM を速打から守る `require-prior-idle-ms` ゲートは全コンボにも (175 ms で) 効くため、mod もコンボもロールや高速連打中は発動しません。

> 以前のリビジョンは修飾キーを全てコンボ (`cb_lshft` / `cb_rctrl` 等) に通し、ホームロウ Mods を使っていませんでした。これら 8 個の修飾コンボは 2026-06 の timeless HRM 再導入時に撤去しています。その後コンボ集合を大幅拡張 (記号・括弧・Ins/Del 派生) し、誤爆耐性のため再チューニングしました。経緯は [キーマップ設計ログ](keymap-design.ja.md) を参照。

## ホームロウ Mods

修飾キーは home 行キーのホールドで取得します (タップ = 文字、ホールド = 修飾キー)。**コンボではありません** — `hm_l` / `hm_r` の振る舞い定義は [設定リファレンス](configuration.ja.md) を参照。

| 左 | Mod | | 右 | Mod |
|----|-----|-|----|-----|
| A | GUI | | J | Ctrl |
| S | Alt | | K | Shift |
| D | Shift | | L | Alt |
| F | Ctrl | | ; | GUI |

`G` は layer-tap (`lt_g 4`、ホールド = Navi)、`H` は素のキー。

`require-prior-idle-ms = 150` ゲートが「timeless」設計の肝です。直前のキー入力から 150 ms 以内に押されたキーは、ホールドしてもタップに倒れます。そのため速いロールや連打では修飾キーが一切出ず、レイテンシゼロ。修飾キーが発動するのは一拍置いた単独ホールドのみです。

### 多重修飾の組み合わせ

複数の home 行キーを同時にホールドして修飾を重ねます：

- **同手の組み合わせ** (例: Ctrl+Shift): 左は F + D、右は J + K をホールド。
- **対角の組み合わせ** (例: Ctrl+Alt): 左右から 1 つずつホールド (例: 左 F + 右 L)。

mod は positional (`hold-trigger-key-positions` で反対手 + 親指のみに発動を限定) なので、home 行キーをホールドしたまま同手の文字を打っても修飾キーではなく文字が出ます。

| Tab    | S + D (13+14) | `&kp TAB` | |
| Delete | U + I (8+9) | `&kp DEL` | Delete 専用コンボ (新規) |

| Insert | O + J (10+20) | `&kp INS` | クロスハンド |
| Enter  | J + ; (20+23) | `&kp ENTER` | 最もスパンが広く timeout が最も緩い (`75`、`COMBO_TERM_WIDE`)。右親指 (位置 44) でも取得可能 |
| Leader | D + F (14+15) | `&df_lead` | 通常はリーダーモード。**Alt 保持中 → Tab** (Alt+Tab)。[リーダーキー](#リーダーキー) 参照 |

### Delete: 専用コンボ *と* mod-morph

Delete を送る方法は **2 通り**になりました：

1. **専用コンボ U + I** → `&kp DEL` を直接発射。
2. **Shift + Backspace コンボ** → `bs_del` mod-morph が Delete を発射。

| 操作 | 結果 |
|------|------|
| U + I | Delete |
| I + O 単独 | Backspace |
| D (Shift) ホールド + I + O | Delete (Shift + Backspace が DEL に morph) |
| K (Shift) ホールド + I + O | Delete |

mod-morph 経路は `config/cornix.keymap` 内の `bs_del` 振る舞いで実現しています。

> 親指 Esc (`&lt 5 ESCAPE`) は素の `&mo 5` に置換しました。親指タップでは Esc が出なくなり、Esc は **W + E コンボのみ**になりました。同じ親指の**ホールド**は従来どおり NumPad (layer 5) を起動します。

## 記号コンボ

各記号は base レイヤー上の **縦ペア** (同じ列で縦に並ぶ 2 キー) で、素の `&kp` に割当てています。Shift やレイヤー切替は不要です。

| 記号 | キー位置 | 記号 | キー位置 |
|------|----------|------|----------|
| `@`  | W + S (1+13)  | `` ` `` | S + X (13+25) |
| `#`  | E + D (2+14)  | `\`     | D + C (14+26) |
| `$`  | R + F (3+15)  | `=`     | F + V (15+27) |
| `%`  | T + G (4+16)  | `~`     | G + B (16+28) |
| `^`  | Y + H (7+19)  | `_`     | H + N (19+33) |
| `+`  | U + J (8+20)  | `-`     | J + M (20+34) |
| `*`  | I + K (9+21)  | `/`     | K + , (21+35) |
| `&`  | O + L (10+22) | `\|`    | L + . (22+36) |

> `~` (G + B) は旧 Space コンボを置換したものです。Space は左親指 (`lt 4 SPACE`) に残っています。

## 括弧・引用符コンボ (Shift 連動 mod-morph)

通常は括弧・引用符を発射し、Shift 保持中は山括弧・波括弧に変化するコンボです (`bs_del` と同じ mod-morph 方式)。振る舞いは `cm_quote` / `cm_lpar` / `cm_rpar` / `cm_lbkt` / `cm_rbkt` ([設定リファレンス](configuration.ja.md) 参照)。

| コンボ | キー位置 | タップ | Shift + コンボ |
|--------|----------|--------|----------------|
| `'`    | J + L (20+22) | `'` | `"` |
| `(`    | J + K (20+21) | `(` | `<` |
| `)`    | K + L (21+22) | `)` | `>` |
| `[`    | M + , (34+35) | `[` | `{` |
| `]`    | , + . (35+36) | `]` | `}` |

> 旧 `k;` (K + ;) のアポストロフィコンボは撤去しました。`'` は J + L へ移行しています。

## コンボ誤爆対策チューニング

拡張したコンボ集合は打鍵に使う文字キーの真上に乗っているため、誤爆耐性を意図的にチューニングしています。2 つの制御は**別軸**で働きます: **不発 (FN)** は **`timeout-ms` を上げて**対処し、**誤発火 (FP)** は **`require-prior-idle-ms`** で抑えます。

- **`require-prior-idle-ms = 175` (COMBO_IDLE)** が全コンボを打鍵流中で無効化します。コンボは一拍置いた後のみ発動します。これが bigram ロール ("io" / "ui" 等) での誤爆を防ぐため、`timeout-ms` の窓は打鍵流中の誤発火を再発させずに緩められます。
- **`timeout-ms` は指のジオメトリで三段階**:
  - **`30` (COMBO_TERM)** — 同指の縦ペア。1 本の指で上下 2 キーを叩くため指間ズレが無く、短い窓で確実に成立します。これらは構造上ロール誤爆にも強い (下記) ため、短い timeout でも誤発火耐性を損ないません。
  - **`60` (COMBO_TERM_HORIZ)** — 別指の横並びペア (Delete U+I、Backspace I+O、Insert O+J、`'` J+L、`(` J+K、`)` K+L、`[` M+,、`]` ,+.)。隣接キーを**別々の指**で押すとズレ (長い中指が先行・小指が遅れる) が出るため、`30` では 2 キーが揃わず不発 (FN) になりました。`60` に緩めています。
  - **`75` (COMBO_TERM_WIDE)** — Enter チョード (J + ;)。最もスパンが広く (index + 小指)、小指が最も遅れるため最も緩い窓です。

構造上の注意 2 点：

- **同指の縦コンボ** (記号の縦ペア: ws / ed / rf / tg / yh / uj / ik / ol と sx / dc / fv / gb / hn / jm / k, / l.) は構造上ロール誤爆に強い (1 本の指は自分の列の 2 キーを跨いでロールできない) ため、短い `30` のままにしています。同じ指で縦ペアを打鍵流中に連続再発火することも通常ないので、誤発火 (FP) にも構造的に強いです。
- 右手ホームの **J / K / L** はコンボの重なりが最も密 (`'` `(` `)` に加え `+ * &` の記号列と J + ; の Enter) で、最大の誤爆リスク帯です。チューニング時はまずここに注目してください。

この三段階は**実機チューニングで決めた現時点の暫定値**で、長期使用での誤発火 (FP) 観察に応じて調整しうる値です。

## リーダーキー

**リーダーキー** (urob の [zmk-leader-key](https://github.com/urob/zmk-leader-key) モジュール。ZMK 公式 `main` 未収録のため west モジュールとして提供) は小さなコマンド namespace を開きます。**Leader** コンボ (D + F) をタップしてリーダーモードに入り、後続のキー列でその割当を発火します。

Leader コンボは `mod-morph` (`df_lead`) です。修飾なしでは `&leader` を実行しますが、**Alt 保持中は Tab** を送出します (`keep-mods` が Alt を残すため Alt+Tab になる)。Alt を保持したまま D + F を連打するとウィンドウを循環できます。

このモジュール版は **モーダル**動作です。timeout / layers / immediate-trigger を持たず、有効な並びが揃うまで待機し、有効なシーケンスに含まれないキーで中断します。

| シーケンス | 動作 | 用途例 |
|------------|------|--------|
| `c` `c` | `&ctrl_ctrl` — Ctrl 二連打 | IDE の "Run Anything" |
| `s` `s` | `&shift_shift` — Shift 二連打 | IDE の "Search Everywhere" |

二連打マクロは `wait-ms = tap-ms = 40` です。host が "double tap" と認識する間隔は IDE 依存のため、ジェスチャが拾われない場合はこの値を調整してください。

**コンボとリーダーの使い分け**: 高頻度の単発操作は、リーダー (最低 2 打鍵) より直結コンボ (1 チョード) のほうが速いです。リーダーは低頻度・多数・ニーモニックなコマンドを namespace にまとめる用途に向きます。

## Symbols レイヤー上の括弧 (代替経路)

上記の括弧・引用符**コンボ**に加え、同じ括弧類は **Symbols レイヤー**上の通常キー (上段と home 行) でも入力できます。コンボが主経路で、Symbols レイヤーはフォールバック / 括弧を持続入力したい場合用に残しています。

| 記号 | Symbols レイヤー上の位置 | 物理キー |
|------|---------------------------|---------|
| `<`  | top 行 pos 0 | Q |
| `>`  | top 行 pos 1 | W |
| `[`  | top 行 pos 2 | E |
| `]`  | top 行 pos 3 | R |
| `{`  | top 行 pos 4 | T |
| `}`  | right top 行 pos 7 | (`` ` `` の隣) |
| `(`  | home 行 | L (Shift+9 と等価) |
| `)`  | home 行 | ; (Shift+0 と等価) |

使い方: 親指 `mo 1` (位置 42) を hold して Symbols を起動し、対応するキーを押す。

## 編集に関する注意

- **ZMK Studio** はレイアウトの閲覧と静的キーマップ編集に使えますが、`bs_del` `hm_l`/`hm_r` レイヤータップ leader などの**振る舞い (behavior) 定義は編集できません**。これらは `config/cornix.keymap` を手で編集して再ビルドする必要があります
- **Keymap Editor** は推奨されません。過去にコンボの key-positions を保存時に勝手に書き換える挙動が確認されており、不整合の原因となります
